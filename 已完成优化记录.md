## 详情

### 1. 后端自动化测试

使用 `mocha` + `supertest` + SQLite 内存数据库，20 个测试用例覆盖 auth(5)、plans(3)、tasks(12)。

运行：`cd server && npm test`

### 2. 错误码标准化

统一为 `{ code, message, details }` 格式。`server/errors.js` 提供 `AppError` 类和 `sendError`/`sendConflict` 函数。10 种错误码覆盖所有端点。

### 3. 移动端日历

`md` 断点以下改为纵向按日列表（含进度条），`md` 以上保持 7 列网格。

### 4. 移动端手势适配

- 日历区域支持水平滑动切换月份（touchstart/touchend 检测，50px 阈值）
- 侧边栏支持从屏幕左边缘右滑打开（<30px 触发，>60px 确认）
- 日历区域支持 ← → 方向键切换月份

### 5. 复制任务日期选择改进

CopyTasksModal 内嵌迷你日期选择网格，用户可直接点击日期选择，同时保留手动 date input 作为补充。选中日期自动排序。

### 6. 快捷键一致性

- PlanList 创建/编辑表单：Escape 取消
- SettingsModal 改密表单：确认密码框 Enter 提交

### 7. 提交物图标 SVG 化

用内联 Heroicons 风格 SVG 替换 TaskItem 中的 emoji：文档图标（📝→document-text）、链接图标（🔗→link）、回形针图标（📎→paper-clip）。

### 13. plans.js 提取 service 层

提取 `services/planService.js`：`getPlansByUser`、`findPlanByName`、`createPlan`、`updatePlan`、`deletePlanWithCleanup`、`getPlanCalendar`。路由层只做请求解析和响应编排。

### 14. TimeRangePicker 组件

抽取 TaskForm 和 TaskPanel 编辑表单的共用时段选择器（HOURS + fmtHour + select JSX）为独立组件，消除 ~20 行重复代码。

### 15. ConflictModeSelector 组件

抽取 BatchFillModal 和 CopyTasksModal 的共用冲突模式选择器（keep_both/skip/overwrite radio）为独立组件，消除 ~30 行重复代码。

### 16. Settings 导入 ZIP 修复

集成 `JSZip` 库。导入时自动检测文件类型：ZIP 文件自动解压提取 `data.json`，JSON 文件直接读取。修复了原代码中 `file.text()` 读 ZIP 二进制文件的 Bug。
